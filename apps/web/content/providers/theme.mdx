## Overview

A comprehensive collection of theme switcher components that enable seamless toggling between light and dark modes. Each variant is carefully crafted with unique UI patterns, smooth animations, and accessibility in mind. Choose from 6 different styles to match your application's design language.

All components are built on top of `next-themes` and feature:
- **Hydration-safe rendering** with loading states
- **Smooth animations** and transitions
- **Screen reader support** with ARIA labels
- **TypeScript** for type safety
- **Zero configuration** - works out of the box

## Required Setup

All theme switcher components require `next-themes` to be configured in your root layout:

### 1. Install next-themes

```bash
npm install next-themes
```

### 2. Configure ThemeProvider
```bash
npminstall
```

### 3. Setup
```tsx
// app/layout.tsx
import { ThemeProvider } from "next-themes";

export default function RootLayout({
  children
}) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}
```

###
- **4 from theme class
- **`attribute="class"`** - Adds `dark` class to html element
- **`defaultTheme="system"`** - Respects OS preference by default
- **`enableSystem`** - Allows system preference detection
- **`disableTransitionOnChange`** - Prevents flash during theme change (optional) Configure Tailwind

###```js
// 3. Configure Tailwind CSStailwind.config.js
module.exports = {
  darkMode: ["class"],
  // ...
}
```

## Troubleshooting

### Hydration Warnings

If you see hydration warnings:
1. Ensure `suppressHydrationWarning` is on `<html>` tag
2. Check ThemeProvider is in root layout
3. Verify all components show loading state when `!mounted`

### Theme Not Changing

1. Check `next-themes` is installed
2. Verify ThemeProvider wraps your app
3. Ensure `attribute="class"` in ThemeProvider
4. Check Tailwind `darkMode: ["class"]` is set

### Icons Not Showing

1. Verify `lucide-react` is installed
2. Check icon imports are correct
3. Ensure icon sizing classes aren't overridden

### System Theme Not Working

1. Use Dropdown or Multi Button variants (they support system)
2. Ensure `enableSystem` is true in ThemeProvider
3. Check browser supports `prefers-color-scheme`

## Advanced Usage

### Programmatic Theme Control

Access theme state directly:

```tsx
"use client";
import { useTheme } from "next-themes";

export function CustomThemeControl() {
  const { theme, setTheme, systemTheme } = useTheme();

  return (
    <div>
      <p>Current: {theme}</p>
      <p>System: {systemTheme}</p>
      <button onClick={() => setTheme("dark")}>Force Dark</button>
    </div>
  );
}
```

### Theme Persistence

`next-themes` automatically persists theme choice to localStorage. To customize:

```tsx
<ThemeProvider
  storageKey="my-app-theme"
  // ... other props
>
```

### Multiple Theme Attributes

Support both class and data attributes:

```tsx
<ThemeProvider
  attribute={["class", "data-theme"]}
  // ... other props
>
```

3. **System preference** - Use Dropdown or Multi Button for power users
4. **Loading states** - Never skip the `!mounted` check
5. **Accessibility** - Test with keyboard and screen readers
