## Overview

Clerk provides a complete authentication solution with beautiful, customizable components for sign-in, sign-up, and user management. Built for modern web applications with a focus on developer experience and security.
`hello`

### Available Implementations

**Three implementation approaches** to fit your needs:

1. **ShadCN Custom** - Fully customizable components using shadcn/ui primitives
2. **Signals (Experimental)** - Advanced components using Clerk's experimental Signals API
3. **Built-in** - Clerk's pre-built components with minimal customization

## ShadCN Custom Components

Fully-featured authentication components built with custom shadcn/ui primitives. Complete control over styling and behavior.

## Signals Components (Experimental)

⚠️ **Experimental**: These components use Clerk's experimental Signals API (`@clerk/clerk-react@5.44.0+`). API may change in future releases.

Advanced components with real-time validation and enhanced state management.

### Sign In (Signals)

Advanced sign-in with experimental Signals API:

**Features:**
- Real-time field validation
- Password reset flow with email code verification
- New password setup after reset
- Enhanced error handling with field-specific errors
- Global error messages
- Loading states with visual feedback
- Back navigation between reset steps

**Installation:**

```bash
bunx shadcn@latest add @elements/clerk-sign-in-signals
```

**Key Differences from ShadCN:**
- Uses `useSignInSignal()` hook for reactive state
- Built-in password reset flow (sendCode → verifyCode → submitPassword)
- Field-level error tracking via `errors.fields`
- Global error handling via `errors.global`

## Required Setup

### 1. Install Clerk

```bash
npm install @clerk/nextjs
```

### 2. Environment Variables

Add to `.env.local`:

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
```

Get your keys from [Clerk Dashboard](https://dashboard.clerk.com).

### 3. Wrap Your App with ClerkProvider

Update your root layout:

```tsx
import { ClerkProvider } from "@clerk/nextjs";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <ClerkProvider>
      <html lang="en">
        <body>{children}</body>
      </html>
    </ClerkProvider>
  );
}
```

### 4. Add Middleware (Recommended)

Protect routes with Clerk middleware:

```bash
bunx shadcn@latest add @elements/clerk-middleware
```

This creates `middleware.ts` in your project root with route protection.

## Advanced Configuration

### OAuth Providers

Configure in Clerk Dashboard → **User & Authentication** → **Social Connections**

Supported providers:
- Google, GitHub, Apple
- Microsoft, Linear, Notion
- Slack, Discord, GitLab
- Spotify, Twitch, Twitter/X

### Multi-Factor Authentication (MFA)

Enable in Clerk Dashboard → **User & Authentication** → **Multi-factor**

Supports:
- TOTP (Authenticator apps)
- SMS codes
- Backup codes

### Waitlist Mode

Enable in Clerk Dashboard → **Configure** → **Restrictions** → **Sign-up mode** → "Waitlist"

Requires waitlist-shadcn component with server actions.

### Custom Redirects

Update redirect URLs in your components:

```tsx
// After successful sign-in/sign-up
router.push("/dashboard"); // Change to your route
```

Or set environment variables:

```bash
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_SIGN_IN_FALLBACK_REDIRECT_URL=/dashboard
NEXT_PUBLIC_CLERK_SIGN_UP_FALLBACK_REDIRECT_URL=/dashboard
```

## Troubleshooting

### Hydration Errors

Add `suppressHydrationWarning` to your HTML tag:

```tsx
<html lang="en" suppressHydrationWarning>
```

### Already Signed In State

Components show "Already Signed In" message when a user is authenticated. Sign out to test:

```tsx
await clerk.signOut();
```

### Waitlist Not Working

1. Verify waitlist mode is enabled in Clerk Dashboard
2. Check `CLERK_SECRET_KEY` is set (required for server actions)
3. Ensure server actions are enabled in Next.js config

### OAuth Redirect Issues

Make sure your OAuth callback URL is configured:

```tsx
redirectUrl: "/elements/clerk/sso-callback"
redirectUrlComplete: "/dashboard"
```

And create the callback page:

```tsx
// app/elements/clerk/sso-callback/page.tsx
import { AuthenticateWithRedirectCallback } from "@clerk/nextjs";

export default function SSOCallbackPage() {
  return <AuthenticateWithRedirectCallback />;
}
```

## Security Best Practices

1. **Never expose `CLERK_SECRET_KEY`** - Keep it server-side only
2. **Enable MFA** - Require for admin/sensitive accounts
3. **Configure rate limiting** - Use Clerk's built-in protection
4. **Use HTTPS** - Always in production
5. **Validate redirects** - Prevent open redirect vulnerabilities
6. **Enable bot protection** - Smart CAPTCHA is enabled by default
7. **Monitor suspicious activity** - Use Clerk's security dashboard
