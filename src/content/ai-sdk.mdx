## Getting Started

### Prerequisites

Before using AI SDK components, you'll need to set up your AI provider. We support OpenAI, Anthropic, Google, and more.

```bash
npm install ai openai
```

### Environment Setup

Add your API keys to `.env.local`:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

### Basic Setup

Create an API route for your AI endpoint:

```typescript
// app/api/chat/route.ts
import { openai } from '@ai-sdk/openai';
import { streamText } from 'ai';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = await streamText({
    model: openai('gpt-4'),
    messages,
  });

  return result.toAIStreamResponse();
}
```

### Tool Calling

The AI SDK components support function/tool calling for structured interactions.

```typescript

import { openai } from '@ai-sdk/openai';

import { z } from 'zod';

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = await streamText({
    model: openai('gpt-4'),
    messages,
    tools: {
      weather: tool({
        description: 'Get the weather in a location',
        parameters: z.object({
          location: z.string().describe('The location to get the weather for'),
```

## Best Practices

1. **Rate Limiting**: Implement proper rate limiting for production use
2. **Error Handling**: Always handle API errors gracefully
3. **Security**: Validate inputs and sanitize outputs
4. **Performance**: Use streaming for better user experience
5. **Monitoring**: Track usage and performance metrics

## Troubleshooting

### Common Issues

**Streaming not working**: Ensure your API route returns `result.toAIStreamResponse()`

**High latency**: Consider using faster models for real-time interactions

**Token limits**: Implement conversation trimming for long chats

### Debug Mode

Enable debug mode to see detailed logs:

```tsx
<AIChat debug={true} apiEndpoint="/api/chat" />
```
